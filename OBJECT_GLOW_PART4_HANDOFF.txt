Subprocess contract used
- Command shape:
  python3 -m itemcutout --input-file <inputPath> --output-file <outputPath> --json
- Implemented with child_process.spawn and argument array (safe, no shell concatenation).
- Captures stdout, stderr, exit code.
- Includes timeout handling (default 120s; env-overridable).

Success handling
- Requires exit code 0.
- Parses stdout as JSON.
- Validates required fields exist:
  - status
  - input_path or inputPath
  - output_path or outputPath
- Normalizes output to camelCase:
  {
    status,
    inputPath,
    outputPath,
    elapsedSeconds,
    inputDimensions,
    outputDimensions,
    glowApplied,
    glowColor
  }

Error handling
- Normalized error object shape:
  {
    code: 'OBJECT_GLOW_PROCESS_FAILED' | 'OBJECT_GLOW_BAD_JSON',
    message,
    exitCode,
    stderr,
    stdoutSnippet,
    inputPath,
    outputPath
  }
- OBJECT_GLOW_PROCESS_FAILED used for spawn/start failures, non-zero exit, timeout, missing input file.
- OBJECT_GLOW_BAD_JSON used for invalid JSON or missing required JSON fields.
- Temporary endpoint returns normalized error payload in response body.

Path ownership assumptions
- Discowarpcore owns path decisions.
- If outputPath is omitted, service computes it:
  - Replaces last path segment named "original" or "originals" with "processed".
  - Output extension forced to .webp.
  - If no original/originals segment exists, output becomes:
    <inputDir>/processed/<inputBaseName>.webp
- Service creates output directory recursively before running objectGlow.

Temporary endpoint added
- POST /api/media/process-test
- Request body:
  {
    "inputPath": "/absolute/path/to/existing/input.jpg",
    "outputPath": "/absolute/path/to/processed.webp" // optional
  }
- Response on success:
  { ok: true, data: <normalized success object> }
- Response on failure:
  { ok: false, error: <normalized error object> }

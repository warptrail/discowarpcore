Batch Import Workflow

Use the Vision Intake TUI for AI-assisted item photo batches. Raw photos go into ~/Intake/inbox, the TUI prepares the batch, Codex fills item JSON, and the TUI validates, packages, and optionally imports directly into Disco Warp Core.

What To Use

Default workflow npm run intake:tui from the repository root.
Input folder Drop raw item photos in ~/Intake/inbox.
Direct import backend Run the backend at http://127.0.0.1:5002.

The old Desktop folders /Users/moonshade/Desktop/item_staging_1 and /Users/moonshade/Desktop/item_staging_2 are deprecated. Do not use them for new batches.

One-Time Setup

  1. Confirm dependencies are installed with npm install from the repository root.
  2. Confirm objectGlow is available. The TUI defaults to ~/Developer/objectiglow.
  3. Start MongoDB and the Disco Warp Core backend when using direct import.
cd /Users/moonshade/Developer/discowarpcore
npm run dev

If objectGlow lives somewhere else, launch the TUI with OBJECTGLOW_REPO=/path/to/objectGlow npm run intake:tui.

Start A New Batch

  1. Copy raw item photos into ~/Intake/inbox.
  2. Open a terminal at /Users/moonshade/Developer/discowarpcore.
  3. Run the TUI.
npm run intake:tui

The TUI shows the current inbox count and any active processing batches.

DISCO WARP CORE VISION INTAKE
Inbox: N images
Processing: active/empty

Choose Start New Batch, then enter:

  • Batch name: human-readable label for this group of photos.
  • Destination location: optional default location.
  • Destination box id: optional default box. Leave empty for orphaned items.
  • Import mode: direct import, export zip only, or validate/package only.

Import Modes

Mode Use When Result
Direct database import The backend is running locally and you want the normal end-to-end path. The TUI uploads the package, validates it through backend intake endpoints, imports it, then archives the local batch in ~/Intake/completed.
Export zip only You want a manual fallback or need to upload through the GUI. The TUI copies a zip to ~/Intake/exports.
Validate/package only, no import You want to inspect the generated package before any import call. The TUI validates and packages inside the processing workspace.

Override the backend URL with DWC_API_BASE=http://127.0.0.1:5002 npm run intake:tui.

What The TUI Does

  1. Moves supported images from ~/Intake/inbox into a new batch raw/ folder.
  2. Runs background removal with scripts/preprocess_vision_images.js.
  3. Writes cleaned transparent cutouts to processed/.
  4. Runs the vision intake builder in init mode to create JSON stubs in item_artifacts/.
  5. Writes CODEX_AGENT_PROMPT.md and copies it to the macOS clipboard when possible.
  6. Pauses while Codex fills the JSON artifacts.
  7. Validates the filled artifacts.
  8. Packages, exports, or imports according to the selected import mode.

Codex Annotation Step

When the TUI pauses, run the AI agent using the prompt it generated:

<batchRoot>/CODEX_AGENT_PROMPT.md

The prompt instructs Codex to inspect cleaned images and edit only JSON files in:

<batchRoot>/item_artifacts/

Codex must not rename, move, delete, or copy images. It must not write to backend media folders or call backend APIs during annotation.

After Codex finishes, return to the TUI and press Enter.

Folder Model

~/Intake/
  inbox/        raw photos waiting to become a batch
  processing/   active resumable batch workspaces
  completed/    successfully imported batch archives
  failed/       reserved for failed batch recovery
  exports/      zip files created for manual GUI upload

A processing batch looks like this:

~/Intake/processing/<batchId>/
  raw/
  processed/
  item_artifacts/
  package/
  logs/
  batch.json
  CODEX_AGENT_PROMPT.md

Raw images are moved into the batch workspace. They are not permanently deleted by the TUI.

Validation Rules

Validation must pass before packaging or import continues. The key invariant is:

image filename basename === JSON imageKey

Example:

processed/IMG_9506.png
item_artifacts/IMG_9506.json

{
  "imageKey": "IMG_9506",
  "sourceFile": "IMG_9506.png",
  "name": "Cordless Drill",
  "description": "Black and yellow cordless drill.",
  "category": "tools",
  "tags": ["drill", "cordless"],
  "quantity": 1
}

Validation logs are written to <batchRoot>/logs/validation.log.

Resume And Recovery

Rerun the TUI and choose Resume Existing Batch for active workspaces under ~/Intake/processing.

  • awaiting_annotation: continues from validation after Codex has filled JSON.
  • packaged: offers direct import or opening the batch folder.
  • failed: shows the last error and offers retry, validation, or open-folder actions.

Use Review Failed Batch to inspect recoverable failures.

Manual GUI Fallback

The Bulk Import screen remains available for manual zip upload, validation review, recovery, provenance inspection, and archived import inspection.

  1. Run a batch with Export zip only.
  2. Open the GUI Bulk Import page.
  3. Upload the zip from ~/Intake/exports.

Reset recovery

Fresh Mac Install Runbook

Use this after wiping the MacBook to reinstall Disco Warp Core, restore local data from Neon Azoth, and verify the Vision Intake TUI plus Neon Azoth control panel TUI still work.

What Survives From Git

Vision Intake TUI
Tracked in git and launched with npm run intake:tui.
Neon Azoth Control Panel
Tracked in git and launched with npm run neonazoth:tui.
Sync Tooling
Database merge, smoke test, and overwrite helpers are tracked in git.

Git does not preserve machine state: backend/.env, SSH keys, MongoDB service data, Homebrew packages, Node dependencies, ObjectGlow installation, and ~/Intake contents must be recreated or restored.

1. Install System Tools

  1. Install Apple command line tools.
xcode-select --install
  1. Install Homebrew if it is not already present.
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  1. Install runtime tools.
brew install node mongodb-community mongodb-database-tools mongosh rsync git
  1. Start MongoDB locally.
brew services start mongodb-community

2. Clone And Install Disco Warp Core

mkdir -p ~/Developer
cd ~/Developer
git clone git@github.com:warptrail/discowarpcore.git
cd discowarpcore
npm run install:all

Create backend/.env

cp backend/example_env.txt backend/.env

Minimum local values:

MONGO_URI=mongodb://127.0.0.1:27017/discowarpcore
PORT=5002
HOST=0.0.0.0
OBJECT_GLOW_REPO=/Users/moonshade/Developer/objectiglow

3. Restore SSH Access To Neon Azoth

If your SSH key and ~/.ssh/config were restored from backup, test first: 

ssh neonazoth 'hostname && whoami'

If key access is missing, run the helper and follow its printed remote instructions:

cd ~/Developer/discowarpcore
scripts/neonazoth/install-codex-ssh-access.sh

After running the remote setup command in an existing Neon Azoth session, verify:

TEST_ONLY=1 scripts/neonazoth/install-codex-ssh-access.sh

4. Restore Local Database And Media From Neon Azoth

This is the full reset recovery path. It overwrites this MacBook's local MongoDB database and local backend/media so the fresh install matches Neon Azoth. 

cd ~/Developer/discowarpcore
npm run neonazoth:tui
  1. Select Overwrite local DB + media from NeonAzoth.
  2. Confirm backup and preview.
  3. Review the rsync --delete dry run.
  4. Type the exact confirmation phrase only when ready: OVERWRITE LOCAL FROM NEONAZOTH.

This option is intentionally destructive. It creates rollback artifacts under dump/, but it still removes local-only records and media from the active app state.

5. Install Or Verify ObjectGlow

The local image processor expects ObjectGlow at /Users/moonshade/Developer/objectiglow unless OBJECT_GLOW_REPO says otherwise.

  • Restore or clone ObjectGlow into ~/Developer/objectiglow.
  • Create or restore its Python virtual environment.
  • Confirm bin/objectglow exists and is executable.
cd ~/Developer/discowarpcore
npm run smoke:objectglow

If ObjectGlow reports a missing rembg backend, install the CPU extra in the ObjectGlow venv: 

~/Developer/objectiglow/.venv/bin/python -m pip install "rembg[cpu]"

6. Verify The Local App

cd ~/Developer/discowarpcore
npm start

Open the app from this Mac or another LAN device:

http://127.0.0.1:5002
http://<macbook-lan-ip>:5002
  • Open an item with an image.
  • Run Retry Process on an image.
  • Confirm the image status becomes completed.
  • Confirm the processed, display, and thumb images render.

7. Verify Vision Intake TUI

mkdir -p ~/Intake/inbox ~/Intake/processing ~/Intake/completed ~/Intake/failed ~/Intake/exports
cd ~/Developer/discowarpcore
npm run intake:tui
  • Confirm the TUI opens and shows inbox/processing counts.
  • Drop test photos into ~/Intake/inbox.
  • Start a small test batch.
  • Use a 3-digit destination box when needed.
  • Choose direct database import only when the backend is running on port 5002.

8. Verify Neon Azoth Control Panel TUI

cd ~/Developer/discowarpcore
npm run neonazoth:tui -- --overwrite-local-check
npm run neonazoth:tui -- --merge-check
npm run neonazoth:tui -- --status
npm run neonazoth:tui
  • Status and health check should show the remote process and health endpoint.
  • Merge Moonshade DB + media into NeonAzoth pushes local-only data into production.
  • Overwrite local DB + media from NeonAzoth restores this MacBook from production.
  • Protected sync + install + build + restart deploys source code to Neon Azoth, preserving media and env files.

9. Quick Recovery Commands

Local restore from Neon Azoth

npm run neonazoth:tui

Choose Overwrite local DB + media from NeonAzoth.

Push local source to Neon Azoth

npm run neonazoth:tui

Choose Protected sync + install + build + restart.

Run image smoke test

npm run smoke:objectglow

Start app

npm start

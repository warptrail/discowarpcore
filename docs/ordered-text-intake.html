Disco Warp Core Intake

Simple Ordered Intake

This is the no-AI intake path for batches where you can type item names faster than an image-recognition workflow could identify them. Drag photos into ~/Intake/simple, run one command to process the images, type names into items.txt, then run one command to upload the batch to Disco Warp Core.

Simple Intake Commands

1. Initialize npm run intake:simple:init processes dragged-in photos with ObjectGlow and creates operator files.
2. Fill names Open ~/Intake/simple/items.txt and type one item name per processed image.
3. Submit npm run intake:simple:submit validates, packages, uploads, and imports the batch.
4. Reset npm run intake:simple:reset archives the batch locally and clears the simple folder.

Pairing is strictly line-number based. The first processed image becomes line 1 in items.txt, the second processed image becomes line 2, and so on. The system does not inspect images or correct ordering.

What Goes In The Folder

The default workspace is ~/Intake/simple. You do not need to type this path when using the normal npm commands. 

~/Intake/simple/
  IMG_1001.HEIC
  IMG_1002.HEIC
  IMG_1003.HEIC

Drag raw item photos directly into that folder. Do not put them inside processed/ or package/. Those are generated folders.

Initialization Step

Run this from the Disco Warp Core repository root:

npm run intake:simple:init

The init command will:

  • Create ~/Intake/simple if needed.
  • Read supported image files directly from ~/Intake/simple.
  • Sort images deterministically.
  • Run every image through ObjectGlow with a midnight background and a randomized glow token.
  • Write compressed WebP files into processed/ using line-number filenames.
  • Create items.txt if it does not exist.
  • Create intake.config.json if it does not exist, including a default batch name.
  • Create pairing_preview.txt so you can review the image order later.
~/Intake/simple/
  IMG_1001.HEIC
  IMG_1002.HEIC
  IMG_1003.HEIC
  processed/
    1.webp
    2.webp
    3.webp
  items.txt
  intake.config.json
  pairing_preview.txt

If processed/ already contains files, init refuses to overwrite it. To intentionally regenerate processed output, run npm run intake:simple:init -- --force.

By default, each image gets a different random glow color while the background stays midnight. To force one fixed glow for the whole batch, pass a token such as arc: 

npm run intake:simple:init -- --glow-token arc

Typing Item Names

Open ~/Intake/simple/items.txt. Type one item name per line. Blank lines are ignored, so do not use blank lines as placeholders. 

Nintendo 64 Controller
Game Boy Advance
USB-C Cable

The number of non-empty names must exactly match the number of files in processed/. For three processed images, enter three item names.

Use pairing_preview.txt and the numbered files in processed/ to keep the text lines in the correct order.

Optional Box Config

Init creates ~/Intake/simple/intake.config.json. Use it to name the batch and, optionally, send every imported item to a specific box. 

{
  "batchName": "simple 2026-05-27 1631",
  "box": "701",
  "apiBase": "http://127.0.0.1:5002"
}
Field Meaning
batchName Optional display name for the imported batch. If omitted, init uses simple plus a timestamp.
box Optional 3-digit box number. Leave it empty to import as no-box/orphaned items.
apiBase Backend API base URL. The default is http://127.0.0.1:5002.

If box is present, it must be exactly three digits, such as 001 or 701. Submit checks that the box exists before uploading anything. If the format is wrong or the box does not exist, the command fails with a terminal message.

Submit Step

Make sure the Disco Warp Core backend server is running, then run: 

npm run intake:simple:submit

The submit command will:

  • Read items.txt.
  • Read processed images from processed/.
  • Validate that name count and image count match exactly.
  • Validate intake.config.json.
  • Preflight the configured box, if one is set.
  • Generate intake.json for review.
  • Generate batch_manifest.json for the existing batch import system.
  • Create a package zip in package/.
  • Upload the package to the backend.
  • Run backend validation.
  • Import the items into the database.

The Import page uses the generated batch manifest. The batch name shown there comes from intake.config.json batchName.

Reset And Archive

After a batch is submitted, run reset to clear ~/Intake/simple for the next batch while preserving the current files under the Disco Warp Core intake archive folder. 

npm run intake:simple:reset

The reset command will:

  • Move the current simple workspace contents into ~/Intake/discowarpcore/simple-archive/.
  • Include raw images, processed/, items.txt, intake.config.json, generated payloads, and package files.
  • Write simple_intake_archive.json in the archived batch folder.
  • Recreate a clean ~/Intake/simple with a blank items.txt and fresh intake.config.json.
~/Intake/discowarpcore/simple-archive/
  2026-05-27_1631_simple-2026-05-27-1631/
    IMG_1001.HEIC
    processed/
      1.webp
    items.txt
    intake.config.json
    intake.json
    batch_manifest.json
    package/
    simple_intake_archive.json

Sorting And Pairing

The init command sorts input photos before processing them. It then writes processed images with numeric names: 1.webp, 2.webp, 3.webp.

Numeric filenames

1.jpg
2.jpg
10.jpg

Camera-style filenames

IMG_1001.HEIC
IMG_1002.HEIC
IMG_1003.HEIC

Best practice: before init, keep photos in the order you want. If needed, rename files to explicit ordered names like 001.jpg, 002.jpg, and 003.jpg.

Generated Files

File or Folder Created By Purpose
processed/ Init ObjectGlow WebP outputs named by pairing line number.
items.txt Init if missing You type one item name per processed image.
intake.config.json Init if missing Batch display name, optional destination box, and backend API base URL.
pairing_preview.txt Init Shows raw photo to processed filename order.
intake.json Submit Readable payload showing item names and image paths.
batch_manifest.json Submit Manifest compatible with the existing batch package import path.
package/ Submit Temporary package folder and zip uploaded to the backend.
simple_intake_archive.json Reset Archive receipt written inside the archived copy of the simple workspace.

Common Failures

Message Fix
No supported image files found Drag photos directly into ~/Intake/simple, then rerun init.
Unsupported file Remove non-image files or convert unsupported image formats before init.
Refusing to overwrite existing output Review processed/, then rerun init with --force if regeneration is intended.
Item count and image count mismatch Make the number of non-empty lines in items.txt match the number of images in processed/.
Invalid box number Use exactly three digits in intake.config.json, such as 701.
Box was not found Create the box first or correct the configured box number.
Backend API is not reachable Start the backend server or set the correct apiBase.
Wrong batch name on Import page Edit batchName in intake.config.json before running submit.

What This Workflow Does Not Do

  • It does not identify objects in photos.
  • It does not read labels or text from photos.
  • It does not infer categories, tags, descriptions, or quantities.
  • It does not correct image order.
  • It does not change the AI-assisted intake workflow.
  • It does not change Simple Text Upload in the frontend.
